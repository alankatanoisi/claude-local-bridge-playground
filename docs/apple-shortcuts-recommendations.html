🧩 Apple Shortcuts × Bridge Runner

Five macOS Shortcuts that wrap bin/local-bridge-runner.js — the local coding-agent loop that talks to Claude via the localhost:11437 bridge (Claude Code OAuth, no Console API key). Each follows the same pattern as your example screenshot: Receive input → Run Shell Script (zsh) → Stop and output.

All examples assume the repo lives at ~/Developer/claude-local-bridge-playground and the bridge is up on http://localhost:11437. Adjust paths to taste.

1 Explain This Code

Share Sheet / Quick Action on any file or folder in Finder → plain-English explanation. read-only

  • Receive Files and Folders from Share Sheet, Quick Actions — if no input: Ask For Input.
  • Run Shell Script (zsh, Pass Input: as arguments):
cd "$HOME/Developer/claude-local-bridge-playground"
TARGET="$1"
node bin/local-bridge-runner.js \
  --cwd "$(dirname "$TARGET")" \
  --allowed-tools list_files,read_file,search_text,glob \
  --max-steps 8 --log-level quiet \
  "Explain what $(basename "$TARGET") does, its key entry points, and anything surprising. Keep it under 300 words."
  • Stop and output Shell Script Result — if nowhere to output: Show Result (or Quick Look).

Read-only tool allowlist means it can never modify the shared file, so --dont-ask isn't even needed.

2 Review My Clipboard

Copy a diff, function, or PR snippet anywhere → instant code review using the built-in review template. read-only hotkey-friendly

  • Get Clipboard → set variable Snippet.
  • Run Shell Script (zsh, Input: Snippet, Pass Input: to stdin):
cd "$HOME/Developer/claude-local-bridge-playground"
SNIPPET="$(cat)"   # clipboard arrives on stdin
node bin/local-bridge-runner.js \
  --bare --template review \
  --allowed-tools read_file \
  --max-steps 4 --max-tokens 1500 --log-level quiet \
  "Review this snippet for bugs, edge cases, and style issues:

$SNIPPET"
  • Show Result — or Copy to Clipboard so the review replaces the snippet, ready to paste into a PR comment.

--bare skips repo context and instruction docs — the snippet itself is the whole context, keeping runs fast and cheap.

3 Ask My Repo (Dictation)

"Hey Siri, Ask My Repo" → dictate a question → get an answer grounded in the codebase. read-only voice

  • Dictate Text (or Ask For Input) → variable Question.
  • Choose from List (optional): pick which project folder to target → variable Project.
  • Run Shell Script (zsh, Input: Question, Pass Input: as arguments):
cd "$HOME/Developer/claude-local-bridge-playground"
node bin/local-bridge-runner.js \
  --cwd "$HOME/Developer/my-project" \
  --include-repo-context --include-repo-map \
  --allowed-tools list_files,read_file,search_text,glob \
  --max-steps 10 --max-cost-usd 0.25 --log-level quiet \
  "$1"
  • Stop and output Shell Script Result — if nowhere to output: Speak Text (Siri reads the answer back).

--include-repo-map gives the model a structural overview up front so voice questions like "where is auth handled?" resolve in fewer tool loops.

4 Fix-It Ticket (Safe Auto-Edit)

Type or share a bug description → runner edits code in an isolated git worktree, then notifies you. writes files worktree-isolated

  • Ask For Input "Describe the bug or small change" → variable Ticket.
  • Run Shell Script (zsh, Input: Ticket, Pass Input: as arguments):
cd "$HOME/Developer/claude-local-bridge-playground"
node bin/local-bridge-runner.js \
  --cwd "$HOME/Developer/my-project" \
  --accept-edits --dont-ask \
  --max-steps 14 --max-cost-usd 1.00 --max-wall-clock-ms 300000 \
  --transcript "$HOME/.bridge-runner/logs/shortcut-fixit.jsonl" \
  --log-level quiet \
  "Work in an isolated worktree (enter_worktree) on this task, make the minimal fix, and report the branch name: $1"
  • Show Notification "Fix-It done" with Shell Script Result.
  • Stop and output Shell Script Result (branch name + summary for review before merging).
⚠️ This is the only write-capable Shortcut here. Shell stays disabled (no --allow-shell), edits land on a throwaway branch via the runner's worktree tools, and hard budget/wall-clock caps bound the blast radius. Review the branch diff before merging.

5 Morning Repo Digest (Automation)

Scheduled Automation (e.g. 9:00 AM) → what changed, what's stale, what to do next → saved to Notes. read-only scheduled session-aware

  • Automation trigger: Time of Day, 9:00 AM, weekdays (Run Immediately).
  • Run Shell Script (zsh):
cd "$HOME/Developer/claude-local-bridge-playground"
node bin/local-bridge-runner.js \
  --cwd "$HOME/Developer/my-project" \
  --session-id daily-digest --resume-session \
  --allowed-tools list_files,read_file,search_text,glob,git_status \
  --template explore \
  --max-steps 10 --max-cost-usd 0.50 --log-level quiet \
  "Compare against what you reported yesterday: summarize git status, recently modified files, TODO/FIXME markers, and suggest the top 3 things to tackle today."
  • Create Note "Repo Digest — Current Date" with Shell Script Result in a "Dev Digests" folder.
  • Show Notification "☕ Your repo digest is ready."

Reusing --session-id daily-digest lets the runner's session store carry yesterday's findings forward, so the digest highlights deltas instead of repeating itself. (First run only: omit --resume-session to seed the session.)

✅ Setup checklist

StepDetail
1. Bridge runningStart the local bridge so http://localhost:11437 answers before any Shortcut runs (consider a LaunchAgent).
2. Node on PATHShortcuts' shell is non-login zsh; if node isn't found, use an absolute path like /opt/homebrew/bin/node.
3. Full Disk AccessGrant Shortcuts access to the project folders it will read via --cwd.
4. Start read-onlyShortcuts 1–3 and 5 use tool allowlists that make writes impossible; graduate to #4 once you trust the flow.
5. Budgets everywhereEvery headless run carries --max-steps plus a cost or wall-clock cap — automations should never run open-ended.
