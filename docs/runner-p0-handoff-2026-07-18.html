What we decided and why

Tool calls are matched as batches, not as FIFO completion

Several tools can run from one assistant response and finish in a different order. Correctness means the following user result message contains exactly one result for every requested ID. Order may differ; missing, duplicate, orphaned, extra, or interleaved IDs may not.

Three fully failed batches trigger a human choice

A four-tool parallel failure counts once. Any success resets the streak. A human permission denial is not a malfunction. After three fully failed batches, the foreground runner offers continue, add guidance, or stop. With no interactive terminal, it stops safely.

Agent and capability profiles were retired

Alan works with granular flags, while profiles created competing sources of authority and fail-open tool behavior. Generic read-only children now receive an explicit seven-tool set. Historical implementations remain under docs/archive/runner-profiles/ for research, not execution.

P0-04 protects both location and file type

--cwd defines where tools may operate. The deny matrix separately defines sensitive material that may never be read, even inside that folder. An agent may someday request another root for explicit user approval, but its own judgment cannot grant authority and credential classes remain denied.

Non-obvious traps

  • Flexible result order does not permit arbitrary message interleaving; batch adjacency remains strict.
  • Do not silently “repair” malformed resumed history. Preserve evidence and fail locally.
  • Do not count parallel failures individually or count a human denial as a broken tool.
  • Do not restore agent/capability profiles to solve a template need.
  • A child can never inherit more authority than its parent.
  • Regex scanning is not an operating-system shell sandbox.
  • --dont-ask does not enable shell.
  • Do not restore /v1/models, OpenAI-compatible routes, API-key fallback, or captured credential replay.
  • Do not claim live acceptance until refreshed credentials support a real multi-turn tool loop.
  • A green documentation check does not currently prove that runner and bridge defaults agree.
  • Preserve historical assessment text; add dated annotations when reality changes.
