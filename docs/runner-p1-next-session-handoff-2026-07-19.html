3. What P1-10 fixed (this session)

Before: a spawn_agent child often ran with a nearly empty command line — quiet log level, default model assumptions, no --no-network, no wall/cost ceilings, weak correlation. The parent saw a text summary and dropped rich usage/settings.

After: the parent builds an inherit bag and the worker CLI gets the same model, effort, thinking, bridge URL, network guard, remaining wall-clock, cost ceiling, temperature, and trace level. Caller tokens move only through an environment variable (never printed on the command line). When the child finishes, the parent stores a childManifest (usage, stop reason, inherited ceilings, tool-effect summary) in the final result and ledger.
